_# A/B Test Idea Library & Hypothesis Framework

This document provides a library of common A/B test ideas and a structured framework for creating strong hypotheses.

## High-Impact A/B Test Ideas

| Element to Test | Common Variations & Ideas |
|---|---|
| **Headline** | - **Benefit-driven vs. Feature-driven**: "Get more leads" vs. "Our tool has AI."
| | - **Clarity vs. Clever**: A straightforward headline vs. a witty one.
| | - **Include a number**: "Join 10,000+ users" vs. "Join our community."
| | - **Question vs. Statement**: "Tired of low conversions?" vs. "We boost conversions." |
| **Call-to-Action (CTA)** | - **Button Copy**: "Get Started" vs. "Create My Account" vs. "Get My Free Trial."
| | - **Button Color**: High-contrast color vs. brand color.
| | - **Placement**: Above the fold vs. below the fold; sticky CTA on scroll.
| | - **Size & Shape**: Larger button, rounded vs. square corners. |
| **Hero Section** | - **Image vs. Video**: A static hero image vs. a short product demo or customer testimonial video.
| | - **Lifestyle vs. Product**: Image of someone enjoying the benefit vs. a screenshot of the product.
| | - **Illustration vs. Real Photo**: Custom graphic vs. a photograph of a person. |
| **Social Proof** | - **Testimonial Placement**: Near the CTA vs. in a dedicated section.
| | - **Logo Bar vs. Case Studies**: A bar of client logos vs. detailed customer stories.
| | - **Quote vs. Video Testimonials**: Text quotes vs. short video clips from customers. |
| **Page Layout** | - **Long-form vs. Short-form**: A detailed page with lots of copy vs. a concise page with minimal text.
| | - **Single Column vs. Two Columns**: A simple, linear layout vs. a multi-column design.
| | - **Order of Sections**: Moving the social proof section higher or lower on the page. |
| **Offer / Value Prop** | - **Free Trial vs. Demo**: "Start your 14-day free trial" vs. "Request a live demo."
| | - **Money-Back Guarantee**: Adding or highlighting a 30-day money-back guarantee.
| | - **Discounting**: "Get 20% off" vs. the standard price. |

## Hypothesis Framework

A strong hypothesis is crucial for a meaningful test. Use this framework:

**Because we observed [DATA/INSIGHT], we believe that [CHANGE] for [AUDIENCE] will result in [OUTCOME]. We will measure this using [METRIC].**

### Example Hypothesis:

- **Observation**: "Because we observed from heatmaps that only 20% of users scroll to the bottom of the page..."
- **Change**: "...we believe that moving the primary CTA button to the hero section..."
- **Audience**: "...for mobile visitors..."
- **Outcome**: "...will result in more clicks on the CTA."
- **Measurement**: "We will measure this using the CTA click-through rate."

**Full Hypothesis**: "Because we observed from heatmaps that only 20% of users scroll to the bottom of the page, we believe that moving the primary CTA button to the hero section for mobile visitors will result in more clicks on the CTA. We will measure this using the CTA click-through rate."
